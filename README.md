# SPRING PLUS
## 13. 대용량 데이터 처리

### 테스트 환경
- 데이터 건수: 500만 건
- 검색 조건: 닉네임 정확히 일치

---

### DB 조회 속도 비교 (execution 시간)

| 방법 | execution 시간 |
|------|:------------:|
| 인덱스 X | 689ms |
| 인덱스 O | 4ms |

---

### API 응답 속도 비교 (Postman 기준)

| 방법 | 첫 조회 | 두 번째 조회 |
|------|:------:|:----------:|
| 인덱스 X, 캐시 X | 726ms | 726ms |
| 인덱스 X, 캐시 O | 726ms | ~9ms |
| 인덱스 O, 캐시 O | 48ms | ~7ms |

---

### 결론
- **인덱스** → DB 풀스캔 제거로 조회 속도 약 172배 향상 (689ms → 4ms)
- **캐시(Caffeine)** → 반복 조회 시 DB 접근 없이 메모리에서 반환 (~9ms)
- **인덱스 + 캐시** → 첫 조회도 빠르고 반복 조회도 빠른 최적의 조합
